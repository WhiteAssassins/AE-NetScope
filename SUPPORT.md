# Support

AE NetScope is a public source-available proprietary project. Support is provided through public project channels when maintainers and contributors have time.

## Before asking for help

- Check `README.md`.
- Check `SECURITY.md` for security-sensitive topics.
- Search existing issues.
- Run the local checks listed in `README.md`.

## Public support

Use GitHub issues for:

- Reproducible bugs.
- Feature requests.
- Documentation problems.

Public support requests must use sanitized information only. Do not post secrets, credentials, real IP inventories, MAC addresses, hostnames, diagrams, screenshots, customer data, or sensitive logs.

## Security reports

Do not report vulnerabilities in public issues.

Follow `SECURITY.md` and use the repository's private security reporting method when configured.

## Scope

Maintainers may not be able to provide:

- Private deployment support.
- Live troubleshooting for production networks.
- Security review of private infrastructure.
- Emergency incident response.
- Help with unsupported forks or heavily modified deployments.

Commercial licensing, private support, marketplace distribution, hosted offerings, reseller rights, OEM use, or paid managed service rights require a separate written agreement with Christopher David Alberto Roque or AE White Devs LLC.
