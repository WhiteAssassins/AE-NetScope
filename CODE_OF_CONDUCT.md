# Code of Conduct

## Purpose

AE NetScope is a public source-available project for sysadmins, network operators, homelab users, and developers building safer LAN inventory tooling.

The project should be a professional, practical, and respectful space. People should be able to report bugs, suggest changes, review code, and discuss operational security without harassment, gatekeeping, or pressure to expose private infrastructure details.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, but accept corrections when impact and intent differ.
- Keep technical criticism focused on the work, not the person.
- Be patient with new contributors and people from different backgrounds.
- Prefer clear explanations over dismissive replies.
- Respect privacy. Do not ask contributors to share real network maps, credentials, public IPs, private IP inventories, MAC addresses, hostnames, screenshots, logs, or configurations unless they are sanitized.
- Treat security-sensitive information carefully and follow `SECURITY.md`.

## Unacceptable behavior

- Harassment, intimidation, stalking, or sustained disruption.
- Insults, personal attacks, slurs, sexualized comments, or discriminatory language.
- Publishing or threatening to publish someone else's private information.
- Sharing credentials, tokens, private keys, real internal network data, exploit details, or sensitive logs in public project spaces.
- Pressuring maintainers or contributors for unpaid support outside reasonable project boundaries.
- Repeatedly ignoring maintainer requests to stop a behavior.
- Any conduct that would make the project unsafe or hostile for contributors.

## Security and privacy rule

This project deals with network inventory data. Public issues, pull requests, discussions, and examples must not include real secrets or sensitive infrastructure details.

Before posting, remove or replace:

- Passwords, API keys, tokens, cookies, private keys, and session IDs.
- Real public IPs, private IP plans, MAC addresses, serial numbers, hostnames, domains, and customer names.
- Screenshots that reveal internal topology, users, device names, or configuration.
- Logs that include credentials, headers, environment variables, or scan results.

If a report cannot be safely sanitized, follow `SECURITY.md` instead of posting publicly.

## Reporting conduct problems

If you see or experience behavior that violates this Code of Conduct:

1. Do not escalate publicly if it puts someone at risk.
2. Contact the project maintainers through the private contact method configured on the repository.
3. If no private contact method is available yet, open a minimal issue asking for a private maintainer contact. Do not include sensitive personal or security details in that issue.

Reports should include:

- What happened.
- Where it happened.
- Links or screenshots if safe to share.
- Whether immediate action is needed.

Maintainers should handle reports with discretion and avoid exposing private information.

## Enforcement

Maintainers may take action to protect the project and its contributors, including:

- Clarifying expectations.
- Editing or removing comments, issues, pull requests, or discussions.
- Temporarily or permanently limiting participation.
- Reporting abuse through the platform's moderation tools.

Enforcement should be proportionate, consistent, and focused on keeping the project safe and productive.

## Scope

This Code of Conduct applies in all project spaces, including issues, pull requests, discussions, documentation, community chat, release threads, and maintainer interactions related to AE NetScope.

It also applies when someone represents the project in public spaces.
